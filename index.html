Introduction

If you've never worked with the MEAN stack before, MeaningfulJS2 is an integrated example of what you can do with MongoDB, Express, Angular 2, and NodeJS.

If you're an experienced MEAN stack developer, MeaningfulJS2 is a springboard for you to quickly customize and refine your way to production.

New to Angular 2?

If you want an easy way to get started with Angular2, this is a great place to begin. Using MeaningfulJS2 will allow you to focus on learning the framework without having to worry about setting up your environment, dealing with dependencies, writing your own build script, etc.

The tutorials written by the Angular team (using TypeScript) are really quite nice. Here are the same Angular 2 tutorials written in JavaScript / ES6.

MeaningfulJS2 currently uses ECMAScript 2015, 6th edition and Babel.

Prerequisites

Here's what you need to have installed before getting started with MeaningfulJS2:

  • NodeJS: You'll need this to run the build scripts and the included Express web server
  • MongoDB: You can mock out the persistence for a while in "development" mode, but you'll eventually need to save your user data somewhere in production
  • JSPM: JSPM (JavaScript Package Manager) manages your client-side dependencies in config.js; NPM (included with NodeJS) manages your server-side dependencies in package.json
  • Gulp: Your build script gulpfile.js is written in Gulp

Getting Started

The best way to get started with MeaningfulJS2 is to install the Yeoman generator and let it guide your way. Once it's installed, spinning up a new MEAN app is as simple as:

  $ mkdir myapp
  $ cd myapp
  $ yo mjs

You can, of course, simply clone the GitHub repo and muscle through it on your own. If you choose this route, don't forget to install the server-side and client-side dependencies locally by typing:

  $ npm install

Build and Run

The gulpfile.js build script comes with several familiar tasks, including clean, build, and run. The easiest way to start your MEAN app and view it in a browser is to type:

  $ gulp clean build
  $ gulp run

If you type gulp without a task, you'll be presented with a list of available tasks:

  $ gulp
  Using gulpfile ~/myapp/gulpfile.js

  Usage
    gulp [TASK] [OPTIONS...]

  Available tasks
    build    Builds runnable website (dev-mode)
    clean    Deletes generated artifacts
    default  *** Default task ***
    dist     Builds production website (prod-mode)
    help     Display this help text.
    run      Runs website (dev-mode)
