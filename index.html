MeaningfulJS is written and maintained by Scott Davis, author of the IBM developerWorks article series Mastering MEAN and the O'Reilly video Architecture of the MEAN Stack.

 

Introduction

MeaningfulJS is a simple, sane MEAN stack.

If you've never worked with the MEAN stack before, MeaningfulJS is an integrated example of what you can do with MongoDB, Express, AngularJS, and NodeJS.

If you're an experienced MEAN stack developer, MeaningfulJS is a springboard for you to quickly customize and refine your way to production.

Prerequisites

Here's what you need to have installed before getting started with MeaningfulJS:

  • NodeJS: You'll need this to run the build scripts and the included Express web server
  • MongoDB: You can mock out the persistence for a while in "development" mode, but you'll eventually need to save your user data somewhere in production
  • Bower: Bower manages your client-side dependencies in bower.json; NPM (included with NodeJS) manages your server-side dependencies in package.json
  • Gulp: Your build script gulpfile.js is written in Gulp

Getting Started

The best way to get started with MeaningfulJS is to install the Yeoman generator and let it guide your way. Once it's installed, spinning up a new MEAN app is as simple as:

$ yo meaningfuljs

You can, of course, simply download a zipfile, a tarball, or clone the GitHub repo and muscle through it on your own. If you choose this route, don't forget to install the server-side and client-side dependencies by typing:

$ npm install
